# 발표 피피티 제작 요령

1. 프로젝트 개요

1-1. 팀원 소개 및 역할
=> 팀원별로 맡은 역할에 대한 기술. 기획, 설계, 구현 등 맡은 일에 대하여 구체적으로 기술하는것이 좋음.

●공동 작업
HTML: 웹 전체 화면 구현
Java Scrip 
 -게임을 시작하면 8개의 같은 글자와 1개의 다른 글자를 게임판 내 랜덤한 위치에 나타나게 한 뒤 하나의 다른 글자를 클릭하면 정답으로 처리하는 기능 구현.
 -정답을 클릭하면 위와 같은 구성이나 글자를 다르게 하여 연속으로 5번 문제를 내고 끝나는 기능 추가.
 -정답 글자를 잠깐 굵어져 돋보이게 하는 힌트 기능이 들어간 버튼 구현

●채여운:
게임 기획 및 시나리오 구성
CSS를 사용하여 웹 화면에 이미지와 색상을 구현
Java Scrip와 CSS로 색 버튼을 클릭하면 웹 화면 전체의 컬러 테마를 변경하는 기능 구현

●최성렬
Java Scrip
 -공동작업으로 구현한 게임을 고도화해 초급(9글자), 중급(49글자), 고급(100글자)의 난이도 기능 추가 구현.
 -초기 화면으로 돌아가는 새로고침 기능이 들어간 버튼 구현
Java Scrip와 CSS로 게임 종료 후 등장하는 애니메이션이 포함된 화면 구현.



1-2. 주제 선정 배경
 => 우리 팀이 왜 이 주제를 선정했는지 간략하게 기술할 것. 

HTML과 CSS, Java Scrip를 종합적으로 활용할 수 있는 구성.
기한 내에 완성할 수 있으면서도 프로그래밍 실력을 향상시킬 수 있을 정도의 난이도.
기획의도가 명확하며 구성이 깔끔하게 드러날 수 있는 구조.
플레이어의 마음을 끌 수 있을 만한 위트와 감성.

1-3. 목표 및 기대효과
 => 제작한 앱을 통해 사용자는 어떤 편익을 누릴 수 있으며, 향후 이 앱을 통해 거둘 수 있는 수익모델이나 기대할 수 있는 효용에 대해 기술 (딱히 지금은 생각이 안나도 이 앱이 업데이트되어서 기능이 많아졌을때를 가정하고 쓰셔도 됩니다.)
간단한 게임 규칙, 속도감 있는 진행, 매력적인 디자인으로 사용자의 기분을 전환시키는 효과가 기대된다.
향후 다양한 난이도 개발과  캐릭터를 추가한다면 이동 시간이나 업무 중 쉬는 시간 등의 틈새 시간에 간단히 할 수 있는 킬링타임 게임으로 발전 가능성이 있다.

1-4. AS IS vs TO BE 
=> 기존 비슷한 타사의 앱과 비교했을 때 우리 앱만의 장점을 기술.  타사 앱의 단점도 같이 기술하면서 개선효과를 적으면 좋음.
